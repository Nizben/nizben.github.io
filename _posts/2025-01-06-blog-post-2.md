When discussing algorithm efficiency, Big O notation is often the go-to metric. In simple words, it provides a high-level understanding of how an algorithm's runtime grows with the input size. However, in the context of modern computing, especially with intricate CPU architectures and caching mechanisms, Big O can sometimes be misleading. This blog post delves into why Big O isn't the only determinant of an algorithm's performance and explores how CPU caches play a fundamental role in real-world applications.

# Understanding Big O notation

Big O notation is a mathematical representation used to describe the upper bound of an algorithm's running time or space requirements in terms of the input size, denoted as N. The idea is to abstract away constants and lower-order terms to only focus on the growth rate as N approaches infinity.

However, while Big O is invaluable for understanding the theoretical scalability of algorithms, it doesn't account for real-world nuances that can significantly impact performance.

## Details of CPU caching

Modern CPUs incorporate multiple layers of cache to bridge the speed gap between the ultra-fast CPU cores and the slower main memory (RAM). These caches store copies of frequently accessed data to reduce latency and improve performance.

### Cache hierarchy

1 - L1 Cache (Level 1):

* Size: Small (e.g., 128KB in M1/M2/M3 chips in new MacBooks for instance).
* Speed: Extremely fast (~3-4 CPU cycles latency).
* Purpose: Stores the most frequently accessed data and instructions.

2 - L2 Cache (Level 2):

* Size: Larger than L1 (e.g., 12MB).
* Speed: Slower than L1 but faster than RAM (~10-20 CPU cycles latency).
* Purpose: Acts as a bridge between L1 and L3 caches, storing less frequently accessed data.

3 - L3 Cache (Level 3) and Beyond:

* Size: Even larger (e.g., up to tens of MBs).
* Speed: Slower than L2 but still faster than RAM.
* Purpose: Shared among multiple CPU cores to facilitate inter-core data sharing.

### Cache Lines and Memory Access

* `Cache Line`: The smallest unit of data transferred between the cache and main memory, typically 64 bytes.
* `Spatial Locality`: If a program accesses a memory location, it's likely to access nearby memory locations soon after.
* `Temporal Locality`: If a program accesses a memory location, it's likely to access the same location again in the near future.

#### Impact on Data Structures:

* `Arrays`: Stored contiguously in memory, leveraging spatial locality. Accessing one element often preloads adjacent elements into the cache.
* `Hash Sets (Hash Tables)`: Elements are scattered across memory, leading to potential cache misses due to non-contiguous access patterns.

## Why Arrays Are Cache-Friendly

Arrays are an essential example of cache-friendly data structures due to their contiguous memory allocation. Here's why:

1 - `Contiguous Memory Allocation:`

`Benefit`: Accessing one element in an array brings adjacent elements into the cache via the same cache line, reducing cache misses.

2 - `Sequential Access Patterns:`

`Benefit`: Iterating over an array sequentially ensures that once a cache line is loaded, multiple elements are accessed without additional memory fetches.

3 - `Prefetching:`

`Benefit`: Modern CPUs can predict and pre-load the next cache lines when sequential access patterns (like arrays) are detected, further minimizing latency.

4 - `Reduced Pointer Overhead:`

`Benefit`: Arrays don’t require pointer dereferencing to access elements, unlike linked lists or hash tables with chaining. This reduces memory accesses and potential cache misses.

#### Example:

```cpp
for (int i = 0; i < N; ++i) {
    process(array[i]);
}
```
* Cache Behavior: Accessing array[0] loads the entire cache line (e.g., 64 bytes, 16 integers). Accessing array[1] to array[15] hits the cache without additional memory fetches.


## When Higher Big O Beats Lower Big O

While Big O provides a theoretical framework, real-world performance is influenced by factors like cache behavior. There are scenarios where an algorithm with a higher Big O complexity can outperform another due to better cache utilization.

### Array Traversal vs. Hash Set Lookups

* Array Traversal (Binary Search):

  * `Complexity`: O(log N)
  * `Cache Behavior`: Highly cache-friendly due to contiguous memory and predictable access patterns.

* Hash Set Lookups:

* Complexity: Average case O(1), worst-case O(N)
  * `Cache Behavior`: Less cache-friendly due to scattered memory accesses and potential cache misses.
  * `Scenario`: For datasets within the L2 cache capacity (~3 million integers), binary search on an array can outperform hash set lookups despite having a higher Big O complexity.

### Naive Matrix Multiplication vs. Strassen's Algorithm


* Naive Matrix Multiplication:

  * `Complexity`: O(N³)
  * `Access Pattern`: Predictable and sequential, benefiting from cache locality.

* Strassen's Algorithm:

  * `Complexity`: Approximately O(N².81)
  * `Access Pattern`: More complex due to recursive partitioning, leading to less cache-friendly behavior.

`Scenario`: For smaller matrices that fit well within the cache (e.g., N ≈ 512), naive multiplication can outperform Strassen's algorithm because of better cache utilization despite its higher asymptotic complexity.

## Benchmarking in action

To illustrate these concepts, let's implement and benchmark the two scenarios discussed: **Array Traversal vs. Hash Set Lookups** and **Naive Matrix Multiplication vs. Strassen's Algorithm**. We'll use C++ for its performance and control over memory management.

### Array vs. Hash Set Benchmark

**Code: array_vs_hashset.cpp**

```cpp
#include <iostream>
#include <vector>
#include <unordered_set>
#include <chrono>
#include <random>
#include <algorithm>

// Function to generate a sorted array of unique integers
std::vector<int> generate_sorted_array(size_t size) {
    std::vector<int> array;
    array.reserve(size);
    for (size_t i = 0; i < size; ++i) {
        array.push_back(static_cast<int>(i));
    }
    // Shuffle to simulate random access patterns
    std::random_device rd;
    std::mt19937 g(rd());
    std::shuffle(array.begin(), array.end(), g);
    // Sort for binary search
    std::sort(array.begin(), array.end());
    return array;
}

// Function to generate a hash set from the array
std::unordered_set<int> generate_hashset(const std::vector<int>& array) {
    std::unordered_set<int> hashset(array.begin(), array.end());
    return hashset;
}

// Binary search on sorted array
bool binary_search_array(const std::vector<int>& array, int key) {
    return std::binary_search(array.begin(), array.end(), key);
}

int main() {
    size_t N = 3'000'000; // 3 million elements to fit into L2 cache
    std::cout << "Generating sorted array with " << N << " elements...\n";
    auto array = generate_sorted_array(N);

    std::cout << "Generating hash set...\n";
    auto hashset = generate_hashset(array);

    // Prepare lookup keys (half existing, half non-existing)
    std::vector<int> keys;
    keys.reserve(N);
    for (size_t i = 0; i < N / 2; ++i) {
        keys.push_back(static_cast<int>(i)); // Existing keys
    }
    for (size_t i = N; i < N + (N / 2); ++i) {
        keys.push_back(static_cast<int>(i)); // Non-existing keys
    }

    // Shuffle keys to simulate random access
    std::random_device rd;
    std::mt19937 g(rd());
    std::shuffle(keys.begin(), keys.end(), g);

    // Benchmark binary search on array
    std::cout << "Starting binary search on array...\n";
    auto start = std::chrono::high_resolution_clock::now();
    size_t found_array = 0;
    for (const auto& key : keys) {
        if (binary_search_array(array, key)) {
            found_array++;
        }
    }
    auto end = std::chrono::high_resolution_clock::now();
    auto duration_array = std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count();
    std::cout << "Array: Found " << found_array << " keys in " << duration_array << " ms.\n";

    // Benchmark hash set lookups
    std::cout << "Starting hash set lookups...\n";
    start = std::chrono::high_resolution_clock::now();
    size_t found_hashset = 0;
    for (const auto& key : keys) {
        if (hashset.find(key) != hashset.end()) {
            found_hashset++;
        }
    }
    end = std::chrono::high_resolution_clock::now();
    auto duration_hashset = std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count();
    std::cout << "Hash Set: Found " << found_hashset << " keys in " << duration_hashset << " ms.\n";

    // Verify results
    if (found_array == found_hashset) {
        std::cout << "Both methods found the same number of keys.\n";
    } else {
        std::cout << "Discrepancy in found keys!\n";
    }

    return 0;
}
```

### Explanation:

1 - **Data Generation:**

Creates a sorted array of 3 million unique integers.
Constructs a std::unordered_set from the array elements.

2 - **Lookup Keys:**

Generates a list of keys where half exist in the dataset and half do not.
Shuffles the keys to simulate random access.

3 - **Benchmarking:**

Measures the time taken for binary search on the array.
Measures the time taken for hash set lookups.

4 - **Verification:**

Ensures both methods found the same number of keys.


### Matrix Multiplication Benchmark

**Code: matrix_multiplication.cpp**

```cpp
#include <iostream>
#include <vector>
#include <chrono>
#include <random>
#include <cassert>

// Function to generate a random square matrix
std::vector<std::vector<int>> generate_matrix(size_t N) {
    std::vector<std::vector<int>> matrix(N, std::vector<int>(N));
    std::random_device rd;
    std::mt19937 gen(rd());
    std::uniform_int_distribution<> dis(0, 10);
    for (size_t i = 0; i < N; ++i)
        for (size_t j = 0; j < N; ++j)
            matrix[i][j] = dis(gen);
    return matrix;
}

// Naive matrix multiplication
std::vector<std::vector<int>> naive_multiply(const std::vector<std::vector<int>>& A,
                                            const std::vector<std::vector<int>>& B) {
    size_t N = A.size();
    std::vector<std::vector<int>> C(N, std::vector<int>(N, 0));
    for (size_t i = 0; i < N; ++i)
        for (size_t j = 0; j < N; ++j)
            for (size_t k = 0; k < N; ++k)
                C[i][j] += A[i][k] * B[k][j];
    return C;
}

// Add two matrices
std::vector<std::vector<int>> add(const std::vector<std::vector<int>>& A,
                                  const std::vector<std::vector<int>>& B) {
    size_t N = A.size();
    std::vector<std::vector<int>> C(N, std::vector<int>(N, 0));
    for (size_t i = 0; i < N; ++i)
        for (size_t j = 0; j < N; ++j)
            C[i][j] = A[i][j] + B[i][j];
    return C;
}

// Subtract two matrices
std::vector<std::vector<int>> subtract(const std::vector<std::vector<int>>& A,
                                       const std::vector<std::vector<int>>& B) {
    size_t N = A.size();
    std::vector<std::vector<int>> C(N, std::vector<int>(N, 0));
    for (size_t i = 0; i < N; ++i)
        for (size_t j = 0; j < N; ++j)
            C[i][j] = A[i][j] - B[i][j];
    return C;
}

// Strassen's matrix multiplication
std::vector<std::vector<int>> strassen_multiply(const std::vector<std::vector<int>>& A,
                                               const std::vector<std::vector<int>>& B) {
    size_t N = A.size();
    if (N == 1) {
        return {{ A[0][0] * B[0][0] }};
    }

    size_t new_size = N / 2;
    // Initialize submatrices
    std::vector<std::vector<int>> A11(new_size, std::vector<int>(new_size)),
                                 A12(new_size, std::vector<int>(new_size)),
                                 A21(new_size, std::vector<int>(new_size)),
                                 A22(new_size, std::vector<int>(new_size)),
                                 B11(new_size, std::vector<int>(new_size)),
                                 B12(new_size, std::vector<int>(new_size)),
                                 B21(new_size, std::vector<int>(new_size)),
                                 B22(new_size, std::vector<int>(new_size));

    // Divide matrices into submatrices
    for (size_t i = 0; i < new_size; ++i) {
        for (size_t j = 0; j < new_size; ++j) {
            A11[i][j] = A[i][j];
            A12[i][j] = A[i][j + new_size];
            A21[i][j] = A[i + new_size][j];
            A22[i][j] = A[i + new_size][j + new_size];

            B11[i][j] = B[i][j];
            B12[i][j] = B[i][j + new_size];
            B21[i][j] = B[i + new_size][j];
            B22[i][j] = B[i + new_size][j + new_size];
        }
    }

    // Compute M1 to M7
    auto M1 = strassen_multiply(add(A11, A22), add(B11, B22));
    auto M2 = strassen_multiply(add(A21, A22), B11);
    auto M3 = strassen_multiply(A11, subtract(B12, B22));
    auto M4 = strassen_multiply(A22, subtract(B21, B11));
    auto M5 = strassen_multiply(add(A11, A12), B22);
    auto M6 = strassen_multiply(subtract(A21, A11), add(B11, B12));
    auto M7 = strassen_multiply(subtract(A12, A22), add(B21, B22));

    // Compute C11, C12, C21, C22
    auto C11 = add(subtract(add(M1, M4), M5), M7);
    auto C12 = add(M3, M5);
    auto C21 = add(M2, M4);
    auto C22 = add(subtract(add(M1, M3), M2), M6);

    // Combine submatrices into result matrix C
    std::vector<std::vector<int>> C(N, std::vector<int>(N, 0));
    for (size_t i = 0; i < new_size; ++i) {
        for (size_t j = 0; j < new_size; ++j) {
            C[i][j] = C11[i][j];
            C[i][j + new_size] = C12[i][j];
            C[i + new_size][j] = C21[i][j];
            C[i + new_size][j + new_size] = C22[i][j];
        }
    }

    return C;
}

int main() {
    size_t N = 128; // Use powers of 2 for simplicity (e.g., 256, 512, 1024)
    std::cout << "Generating two " << N << "x" << N << " matrices...\n";
    auto A = generate_matrix(N);
    auto B = generate_matrix(N);

    // Benchmark Naive Multiplication
    std::cout << "Starting naive matrix multiplication...\n";
    auto start = std::chrono::high_resolution_clock::now();
    auto C_naive = naive_multiply(A, B);
    auto end = std::chrono::high_resolution_clock::now();
    auto duration_naive = std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count();
    std::cout << "Naive Multiplication Time: " << duration_naive << " ms.\n";

    // Benchmark Strassen's Multiplication
    std::cout << "Starting Strassen's matrix multiplication...\n";
    start = std::chrono::high_resolution_clock::now();
    auto C_strassen = strassen_multiply(A, B);
    end = std::chrono::high_resolution_clock::now();
    auto duration_strassen = std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count();
    std::cout << "Strassen's Multiplication Time: " << duration_strassen << " ms.\n";

    // Optional: Verify that both methods produce the same result
    bool correct = true;
    for (size_t i = 0; i < N && correct; ++i) {
        for (size_t j = 0; j < N && correct; ++j) {
            if (C_naive[i][j] != C_strassen[i][j]) {
                correct = false;
                std::cout << "Mismatch at (" << i << ", " << j << "): "
                          << C_naive[i][j] << " vs " << C_strassen[i][j] << "\n";
            }
        }
    }
    if (correct) {
        std::cout << "Both multiplication methods produced the same result.\n";
    } else {
        std::cout << "Discrepancy found between multiplication methods!\n";
    }

    return 0;
}
```

### Explanation:

1 - **Matrix Generation:**

Creates two random N x N matrices with integer elements.

2 - **Naive Multiplication:**

Implements the straightforward triple-loop matrix multiplication.

3 - **Strassen's Multiplication:**

Recursively divides matrices into submatrices and combines the results based on Strassen's method.
Note: This implementation is basic and does not include optimizations like switching to naive multiplication for small submatrices, which is common in practical Strassen implementations to reduce overhead.

4 - **Benchmarking:**

Measures the time taken by each multiplication method using std::chrono.

5 - **Verification:**

Optionally checks if both methods produce identical results to ensure correctness.
